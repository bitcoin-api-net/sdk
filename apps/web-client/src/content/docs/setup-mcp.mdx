The Bitcoin API ships an **MCP (Model Context Protocol) server** so any
MCP-compatible AI tool — Cursor, VS Code (Copilot), Claude Code, Claude Desktop,
Windsurf, Zed, Codex CLI, Gemini CLI, … — can search our docs, fetch full
pages, look up code recipes, and inspect the OpenAPI schema while it writes
your code.

The endpoint is **public, read-only and free**. No API key is required.

## Server details

- **Name**: `bitcoin-api-docs`
- **URL**: `https://api.bitcoinapi.dev/mcp`
- **Transport**: Streamable HTTP

## Manual setup

Most tools accept a common JSON config. If your tool isn't listed below, try
adding this to its MCP settings:

```json
{
  "mcpServers": {
    "bitcoin-api-docs": {
      "type": "http",
      "url": "https://api.bitcoinapi.dev/mcp"
    }
  }
}
```

### Cursor

Add to `~/.cursor/mcp.json` (or `.cursor/mcp.json` in your project):

```json
{
  "mcpServers": {
    "bitcoin-api-docs": {
      "url": "https://api.bitcoinapi.dev/mcp"
    }
  }
}
```

Then restart Cursor. The agent will auto-discover the tools.

### VS Code (GitHub Copilot)

Either click the **Install MCP** button above, or add to your user
`settings.json`:

```json
{
  "mcp": {
    "servers": {
      "bitcoin-api-docs": {
        "type": "http",
        "url": "https://api.bitcoinapi.dev/mcp"
      }
    }
  }
}
```

### Claude Code

```bash
claude mcp add --transport http bitcoin-api-docs https://api.bitcoinapi.dev/mcp
```

### Claude Desktop

1. Open **Settings → Connectors → Add custom connector**.
2. Set name to `bitcoin-api-docs`.
3. Set URL to `https://api.bitcoinapi.dev/mcp`.

### Windsurf

Windsurf doesn't speak Streamable HTTP yet, so use a local proxy. Add to
`~/.codeium/windsurf/mcp_config.json`:

```json
{
  "mcpServers": {
    "bitcoin-api-docs": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://api.bitcoinapi.dev/mcp"]
    }
  }
}
```

### Zed

Add to `~/.config/zed/settings.json`:

```json
{
  "context_servers": {
    "bitcoin-api-docs": {
      "settings": {},
      "enabled": true,
      "url": "https://api.bitcoinapi.dev/mcp"
    }
  }
}
```

### Codex CLI

Add to `~/.codex/config.toml`:

```toml
[mcp_servers.bitcoin-api-docs]
command = "npx"
args = ["-y", "mcp-remote", "https://api.bitcoinapi.dev/mcp"]
```

### Gemini CLI

Add to `~/.gemini/settings.json`:

```json
{
  "mcpServers": {
    "bitcoin-api-docs": {
      "httpUrl": "https://api.bitcoinapi.dev/mcp"
    }
  }
}
```

## Available tools

Once connected, your agent gets six tools:

### `get_docs_list()`

Returns every documentation chunk with `url`, `anchor` and `title`. Use this
to discover what's available, then call `get_doc` to read a page in full.

### `get_doc(url)`

Returns the full markdown of a documentation page by its URL/path
(e.g. `/docs/quickstart`).

### `get_api_endpoints_list()`

Returns all available API endpoints with `operationId`, `method`, `path`,
`summary`, `description` and `tags`.

### `api_endpoint(operationId)`

Returns the raw OpenAPI operation object (parameters, request/response
schemas) for the given `operationId` — e.g. `getCurrentPrice`.

### `get_recepies_for_endpoint(operationId)`

Returns all code recipes that use a given endpoint, with `url`, `title`,
`description` and `language`.

### `get_recipe(url)`

Returns the full markdown of a recipe by its URL/path, plus metadata
(`language`, `tags`, `endpoints`, `runUrl`, …).

## Try it

Once configured, ask your agent something like:

> _"Using the bitcoin-api-docs MCP, show me how to subscribe to live BTC/USDT
> prices over WebSocket with Bitcoin API."_

The agent will call `get_api_endpoints_list` → `api_endpoint` →
`get_recepies_for_endpoint` → `get_recipe` and produce a working snippet
grounded in the current docs.

## Troubleshooting

- Make sure your tool supports **Streamable HTTP** transport. If not, use
  the `mcp-remote` proxy snippets above (Windsurf / Codex CLI).
- Double-check the URL: `https://api.bitcoinapi.dev/mcp`.
- Restart your editor after editing config files.
